# Annotating Screenshots

DrawShare lets you place an image on the canvas and draw directly over it — highlighting areas of interest, adding labels, circling bugs, or marking up design mockups. Export the result as a PNG when you are done.

> **Note:** Image import via paste and drag-and-drop is an upcoming feature. This page describes the intended workflow and will be updated when the feature ships.

## Workflow

### Import an Image

**Paste (upcoming):**

1. Copy an image to your clipboard (screenshot tool, browser right-click, etc.).
2. Click the DrawShare canvas and press `Ctrl+V` / `Cmd+V`.
3. The image is placed on the canvas and locked as a background layer.

**Drag-and-drop (upcoming):**

1. Drag an image file (PNG, JPG, WebP) from your file manager.
2. Drop it onto the DrawShare canvas.
3. The image is placed and ready to annotate.

### Annotate

Use any DrawShare drawing tool over the imported image:

- **Pen** — precise marks, signatures, freehand circles
- **Highlighter** — semi-transparent colour blocks to draw attention to regions
- **Text** — labels and callouts

### Export

When your annotation is complete:

1. Open the page menu.
2. Choose **Export PNG**.
3. The downloaded file contains both the original image and your annotations composited together.

## Example Uses

| Context | What to annotate |
|---------|-----------------|
| Bug report | Circle the broken UI element, add a text label |
| Design review | Mark up a mockup with colour-coded feedback |
| Document review | Highlight key passages, add margin notes |
| Map or diagram | Draw a route or region boundary over a screenshot |
| Code review on screen | Circle the suspect line in a screenshot |

## Tips

- Keep annotation strokes on separate pages from your original canvas work so you can toggle between them.
- Use a distinctive colour for annotations so they stand out from the underlying image.
- Export immediately after annotating — canvas data is local, but PNG export gives you a portable, shareable file.
